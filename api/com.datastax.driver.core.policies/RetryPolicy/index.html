interface RetryPolicy

A policy that defines a default behavior to adopt when a request returns a TimeoutException or an UnavailableException. Such policy allows to centralize the handling of query retries, allowing to minimize the need for exception catching/handling in business code.

Implemented By

  • DefaultRetryPolicy
  • DowngradingConsistencyRetryPolicy
  • FallthroughRetryPolicy
  • LoggingRetryPolicy

Methods

RetryDecision onReadTimeout (Statement statement, ConsistencyLevel cl, int requiredResponses, int receivedResponses, boolean dataRetrieved, int nbRetry)

Defines whether to retry and at which consistency level on a read timeout.

Note that this method may be called even if requiredResponses >= receivedResponses if dataPresent is false (see ReadTimeoutException#wasDataRetrieved).

Parameters:
Name Type Details
statement Statement

the original query that timed out.
cl ConsistencyLevel

the original consistency level of the read that timed out.
requiredResponses int

the number of responses that were required to achieve the requested consistency level.
receivedResponses int

the number of responses that had been received by the time the timeout exception was raised.
dataRetrieved boolean

whether actual data (by opposition to data checksum) was present in the received responses.
nbRetry int

the number of retry already performed for this operation.
Returns:
Type Details
RetryDecision

the retry decision. If RetryDecision.RETHROW is returned, a ReadTimeoutException will be thrown for the operation.
